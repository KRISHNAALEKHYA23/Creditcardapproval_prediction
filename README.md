# Credit Card Approval Prediction System

A machine-learning-powered Flask web application that predicts whether a
credit card application will be approved or rejected, based on an
applicant's financial and demographic profile.

## Project structure

```
ccap/
├── data/
│   ├── generate_data.py        # creates synthetic application_record.csv / credit_record.csv
│   ├── application_record.csv  # applicant demographic & financial data
│   └── credit_record.csv       # monthly payment status history
├── notebooks/
│   └── credit_card_approval_prediction.ipynb   # full EDA + modeling notebook
├── model/                      # generated by train_pipeline.py
│   ├── best_model.pkl
│   ├── scaler.pkl
```

## 1. Setup

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 2. Get a dataset

This repo ships with `data/generate_data.py`, which creates a **synthetic**
dataset that mirrors the exact schema of the well-known Kaggle "Credit Card
Approval Prediction" dataset (`application_record.csv` + `credit_record.csv`).
That means the whole pipeline below runs out of the box with no external
downloads.

If you'd rather use the real dataset:
1. Download `application_record.csv` and `credit_record.csv` from the
   Kaggle dataset "Credit Card Approval Prediction" (search that title on
   kaggle.com — search results change over time so it's not linked directly here).
2. Drop both files into the `data/` folder, overwriting the synthetic ones.
3. Re-run `train_pipeline.py` — everything downstream (encoders, scaler,
   model, Flask app) automatically adapts to the real data.

To regenerate the synthetic data yourself:
```bash
cd data
python generate_data.py
```

## 3. Train the models

```bash
python train_pipeline.py
```

This will:
- Merge `application_record.csv` with a risk label derived from
  `credit_record.csv` (STATUS 2–5 → high risk → reject)
- Clean data (missing values, duplicates) and engineer features
  (age in years, years employed, etc.)
- Generate EDA plots into `static/plots/`
- Train **Logistic Regression, Decision Tree, Random Forest, and XGBoost**
- Print accuracy / confusion matrix / classification report for each
- Save the best-performing model + its scaler/encoders into `model/`

You can also open and run `notebooks/credit_card_approval_prediction.ipynb`
in Jupyter for the same pipeline with inline plots and commentary.

## 4. Run the web app

```bash
python app.py
```

Visit `http://127.0.0.1:5000`:
- **`/`** — project overview and live model-comparison table
- **`/predict`** — applicant intake form
- form submission → instant Approved / Declined result with confidence %

## 5. Deploying to IBM Watson Machine Learning (cloud hosting)

The project brief calls for an IBM Watson ML deployment pipeline so the
model can serve predictions from the cloud instead of (or alongside) the
local Flask app. Broad strokes:

1. **Create an IBM Cloud account** and provision a **Watson Machine
   Learning** service instance, plus a **Watson Studio** project to hold
   the notebook/model.
2. **Install the WML Python client** in your environment:
   ```bash
   pip install ibm-watson-machine-learning
   ```
3. **Authenticate and get a client**:
   ```python
   from ibm_watson_machine_learning import APIClient

   wml_credentials = {
       "url": "https://us-south.ml.cloud.ibm.com",   # region-specific
       "apikey": "<YOUR_IBM_CLOUD_API_KEY>"
   }
   client = APIClient(wml_credentials)
   client.set.default_space("<YOUR_DEPLOYMENT_SPACE_ID>")
   ```
4. **Store and deploy the trained model** (using the same `best_model`
   object produced by `train_pipeline.py`):
   ```python
   sofware_spec_uid = client.software_specifications.get_id_by_name("runtime-23.1-py3.10")

   model_meta = {
       client.repository.ModelMetaNames.NAME: "credit_card_approval_model",
       client.repository.ModelMetaNames.TYPE: "scikit-learn_1.1",
       client.repository.ModelMetaNames.SOFTWARE_SPEC_UID: sofware_spec_uid
   }
   model_details = client.repository.store_model(
       model=best_model, meta_props=model_meta, training_data=X_train, training_target=y_train
   )
   model_uid = client.repository.get_model_id(model_details)

   deployment_meta = {
       client.deployments.ConfigurationMetaNames.NAME: "credit-card-approval-deployment",
       client.deployments.ConfigurationMetaNames.ONLINE: {}
   }
   deployment = client.deployments.create(model_uid, meta_props=deployment_meta)
   ```
5. **Get the scoring endpoint** and call it from the Flask app (or any
   client) instead of loading the local pickle file:
   ```python
   scoring_url = client.deployments.get_scoring_href(deployment)
   payload = {"input_data": [{"fields": FEATURE_COLUMNS, "values": [[...]]}]}
   response = client.deployments.score(deployment_uid, payload)
   ```
6. In `app.py`, you can swap the local `model.predict(...)` call for a
   request to `scoring_url` so the Flask app becomes a thin client over the
   cloud-hosted model — useful for scaling predictions across multiple
   servers without redistributing the pickle file.

> Note: steps 1–4 require an actual IBM Cloud account, API key, and
> deployment space ID, which only you can provision. The code above is the
> template to fill in with those credentials.

## Notes on the synthetic dataset

`TARGET` is derived the same way the real dataset is typically framed:
an applicant's monthly `STATUS` history (0–5, C, X) is reduced to a binary
label — any month at 60+ days past due (`STATUS` 2–5) marks that applicant
high-risk (`TARGET=1`, decline). Everything else is `TARGET=0` (approve).
Swap in the real Kaggle CSVs at any time; no other code changes are needed.
